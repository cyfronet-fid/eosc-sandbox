Core Innovation Sandbox

For End-Users For Pilot Nodes and Resource Providers
The EOSC Core Innovation Sandbox by EOSC Beyond is a pre-production environment of the EOSC Platform that acts as a testing and staging environment - equipped with advanced automation tools - for EOSC Core service providers, EOSC Exchange service providers and EOSC Nodes in the integration process with EOSC.
The Sandbox can be used by EOSC Nodes and other providers for the technical deployment of their use cases in EOSC.
It can also be used by researchers and other end-users to assess the EOSC capabilities.

Main features

Federation Capabilities

EOSC EU Node: the Sandbox offers for integration the same federating capabilities delivered by the EOSC EU Node (e.g. the AAI Federation, the Federated Resource Catalogue, the Helpdesk, the Application Workflow Management, etc.) with compatible interfaces for testing.

Additional Federating Capabilities developed by the project.

Technical support

Assist providers willing to integrate with EOSC with dedicated technical support and offer tools to automate the integrations (e.g. easy-to-use connectors in the form of software libraries).

Showcase new EOSC Core capabilities

Research communities and scientists can test and assess the new EOSC capabilities that EOSC Beyond delivers.

For Pilot EOSC Nodes and other providers

If you are representing a potential new EOSC Node or a service provider, you can use the EOSC Innovation Sandbox to integrate your Node or service with EOSC and test and validate your work before moving into production.
A team of technical experts is available to support you in this process. Please, contact [WP15/WP16 support mailing list] shortly describing your case and the type of integration you would like to achieve (e.g. setting up a new EOSC node for the structural biology community, publishing data in EOSC, offering cloud resources to EOSC, etc.).

Currently, the following federating capabilities are offered for integration:

Order Management
• Offer a framework for providers to define offers and a unique interface for end-users to request access to resources Execution Framework
Monitoring
• Provide information about the availability and quality of EOSC services (Core and Exchange)
Resource Catalogue
& Marketplace
• EOSC Resource Catalogue (services, datasets and other research products) exposed to the end-user via the Marketplace
• Advanced search engine
• Research communities can publish (push) and retrieve (pull) resources to/from the federated catalogue
Helpdesk
• EOSC Helpdesk as entry point to request support for all EOSC services
Order Management
• Provide information about usage of services and resources in EOSC (Core and Exchange)
AII
• Single sign-on over federated infrastructures
