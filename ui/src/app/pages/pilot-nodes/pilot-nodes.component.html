For Pilot EOSC Nodes and other providers

If you are representing a potential new EOSC Node or a service provider, you can use the EOSC Innovation Sandbox to integrate your Node or service with EOSC and test and validate your work before moving into production.
A team of technical experts is available to support you in this process. Please, contact us shortly describing your case and the type of integration you would like to achieve (e.g. setting up a new EOSC node for the structural biology community, publishing data in EOSC, offering cloud resources to EOSC, etc.). Contact us

Currently, the following federating capabilities are offered for integration:

Order Management
• Offer a framework for providers to define offers and a unique interface for end-users to request access to resources Execution Framework
Monitoring
• Provide information about the availability and quality of EOSC services (Core and Exchange)
Integration Suite
• The EOSC Integration Suite provides tools designed to share and reuse software libraries (adapters) among research communities and technology providers, addressing common patterns in scientific applications. Add Messaging to the list of offered services
Resource Catalogue
& Marketplace
• EOSC Resource Catalogue (services, datasets and other research products) exposed to the end-user via the Marketplace
• Advanced search engine
• Research communities can publish (push) and retrieve (pull) resources to/from the federated catalogue
Helpdesk
• EOSC Helpdesk as entry point to request support for all EOSC services
Messaging
• Enable efficient communication between applications with advanced EOSC messaging service.
Accounting
• Provide information about usage of services and resources in EOSC (Core and Exchange)
AAI
• Single sign-on over federated infrastructures
Execution Framework
• The EOSC Execution Framework provides an extendable suite of composability-based applications designed to enhance the deployment and accessibility of services within the EOSC ecosystem.
